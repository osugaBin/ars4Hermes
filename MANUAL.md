# ARS4Hermes 使用指南

## Step 1: deep-research — 学术深度研究

### 触发方式

在对话中直接说出触发词即可：

| 你说 | 模式 | 行为 |
|------|------|------|
| `开始深度研究 人工智能对教育的影响` | full | 检索 10-15 个来源，输出 3000-8000 字完整报告 |
| `快速调研 区块链在教育中的应用` | quick | 检索 5-6 个核心来源，输出 500-1500 字简报 |
| `系统综述 AI辅助评估的有效性` | systematic-review | 按 PRISMA 流程（PICOS → 检索→ 筛选 → 偏倚评估 → 综合） |
| `引导我研究 高等教育质量保障` | socratic | 5 层苏格拉底对话，帮你厘清研究问题 |
| `事实核查 中国大学数量在下降` | fact-check | 3-5 个来源交叉验证，输出裁决 |
| `文献综述 SDGs在高等教育中的实施` | lit-review | 8-12 个来源的主题综合 + 证据矩阵 |

### 工作流程（自动执行）

1. **Phase 0: 模式派发** — 根据触发词自动选择模式
2. **Phase 1: 检索与提取** — 3-5 个不同角度的搜索查询 → 筛选 → 提取关键信息
3. **Phase 2: 交叉验证** — 共识/矛盾/孤立声明分类，证据层级检查
4. **Phase 3: Integrity Gate** — 5 种 AI 失败模式检查：
   - 捏造引用？方法捏造？捷径依赖？框架锁定？幻觉数据？
5. **Phase 4: 输出报告** — 结构化报告含执行摘要、共识事实、矛盾点、参考文献

### 关键规则

- **每个结论必须标注来源 URL** — 不允许无来源的主张
- **完整性警告** — 发现任何问题用中文红色标记在报告开头
- **AI 声明** — 所有报告开头必须披露 AI 辅助
- **证据层级** — A 级（元分析/RCT）> B 级（队列研究）> C 级（案例研究）> D 级（意见）
- **预测期刊排除** — 无编委会、72h 录用、非 Scopus 索引的期刊排除

### Socratic 模式使用技巧

- 适合：有模糊兴趣但不确定具体研究方向
- 不适合：已经有明确研究问题（应直接使用 full 模式）
- 5 层推进：问题框架 → 方法反思 → 证据设计 → 批判自省 → 意义
- 每层至少 2 轮对话，用户随时可以要求跳过
- 10 轮未收敛 → 建议切换到 full 模式
- 15 轮 → 自动汇总 INSIGHTs 结束

### 故障处理

| 问题 | 自动处理 |
|------|----------|
| 来源不足（< 5 个） | 扩展搜索（同义词、放宽年限、邻近学科） |
| 所有来源质量低 | 标记所有结论为「有保留」 |
| 来源提取失败 | 自动回退到浏览器工具 |
| 交叉验证无共识 | 作为矛盾点呈现，列出双方证据 |

### 输出示例

完整报告会自动保存为 `deep-research_<主题>_<模式>_<日期>.md`。

## Step 2: academic-paper — 学术论文写作

### 触发方式

| 你说 | 模式 | 行为 |
|------|------|------|
| `写学术论文 [主题]` | full | 从大纲到完整论文（先确认论文类型，再逐章写作） |
| `写综述 [主题]` | full (文献综述模板) | 主题综合式文献综述 |
| `修改论文` 并提供草稿+审稿意见 | revision | 逐条处理 reviewer 意见，生成修订跟踪表 |
| `生成大纲 [主题]` | outline-only | 只生成章节结构，不写正文 |
| `写摘要 [主题]` | abstract-only | 生成中英文双语摘要 |

### 工作流程（自动执行）

1. **Phase 0: 模式派发 & 论文类型确认**
   - 询问：实证研究(IMRaD)/文献综述/理论分析/案例研究/政策简报
   - 确认目标期刊/会议、字数要求

2. **Phase 1: 文献基础检查**
   - 如果有 deep-research 报告 → 基于它写作
   - 如果没有 → 要求先运行 deep-research 或提供来源
   - 遵循 Anti-Leakage Protocol：只从提供的材料写作，不捏造

3. **Phase 2: 逐章写作**
   - 先生成大纲供确认
   - 建议 3-5 个标题选项
   - 逐节写作，每句标注来源引用
   - 材料不足处标记 `[MATERIAL GAP]`

4. **Phase 3: 引用合规**
   - 默认 APA 7th（含中文引用规则）
   - 每个引用必须有位置锚点（quote/page/section）
   - 不可验证的引用 → 移除并标记

5. **Phase 4: 双语摘要**
   - 英文 150-250 字 + 中文 200-300 字

6. **Phase 5: 诚信披露**
   - 自动添加 AI 使用声明
   - 完成写作质量自查清单

### 关键规则

- **Data access: verified_only** — 写作阶段不主动联网搜索
- **Anti-Leakage** — 每个事实主张必须追溯至提供的来源
- **Anchor requirement** — 每个引用必须有位置锚点
- **[MATERIAL GAP]** — 材料不足时标记，不从记忆填充
- **Methods section** — 只能写用户实际提供的方法

### 文件结构

```
~/.hermes/skills/research/academic-paper/
├── SKILL.md                        (主定义)
├── references/
│   ├── paper_structure_patterns.md (6 种论文结构模板)
│   ├── anti_leakage_protocol.md    (知识隔离协议)
│   └── apa7_chinese_citation_guide.md (APA7 中文引用规则)
└── templates/
    ├── bilingual_abstract_template.md (双语摘要模板)
    └── revision_tracking_template.md  (修订跟踪表)
```

## Step 3: academic-paper-reviewer — 多视角同行评审

### 触发方式

| 你说 | 模式 | 行为 |
|------|------|------|
| `评审论文` 并附上论文文本或文件路径 | full | 7 维度评分 + 详细意见 + 引用完整性核查 |
| `审稿 ~/paper.md` | full | 接收文件路径，输出完整审稿报告 |
| `快速评估 ~/paper.md` | quick | 总体评价 + 3 个最主要问题 |
| `引导改进 ~/paper.md` | guided | 逐节交互式审稿 + 即时修改建议 |

### 工作流程（自动执行）

1. **Phase 0: 论文读取**
   - 读取文件或粘贴的文本
   - 提取标题、摘要、各章节、参考文献

2. **Phase 1: 7 维度评审**（仅 full 模式）

   | 维度 | 权重 | 评分 1-5 |
   |------|------|----------|
   | 原创性与创新性 | 15% | 新观点/方法/数据？ |
   | 方法论严谨性 | 25% | 方法合理、可复现？ |
   | 论证与证据 | 20% | 论点有充分证据支撑？ |
   | 论证连贯性 | 15% | 逻辑流畅、结构合理？ |
   | 清晰度与结构 | 10% | 语言表达、图表清晰？ |
   | 文献覆盖 | 10% | 引用关键文献？ |
   | 影响力与意义 | 5% | 对该领域贡献？ |

   加权总分 → 决定：Accept / Minor Revision / Major Revision / Reject

3. **Phase 2: 问题分类**
   - 🔴 **必须修改** — 影响学术有效性
   - 🟡 **建议修改** — 值得改进但不致命
   - 🟢 **锦上添花** — 小建议

4. **Phase 3: 引用完整性核查**（仅 full 模式）
   - 抽查 5 个引用
   - 检测：无锚点/捏造引用/过度便利引用
   - 输出「可疑引用清单」

5. **Phase 4: 反奉承机制**
   - 当用户质疑评审意见时生效
   - 对质疑评分（1-5），≥4 才可让步
   - 禁止连续让步两次以上
   - 坚持不等于不礼貌，压力不是证据

### 关键规则

- **Data access: redacted** — 仅基于论文文本，不主动联网
- **反奉承核心**：评分 ≤3 的质疑 → 维持原意见，不软化语言
- **引用核查**：HIGH-RISK 发现 → 推荐拒稿或深度验证
- **引导模式**：逐节交互，读完一节就提问，不一次性输出完整报告

### 文件结构

```
~/.hermes/skills/research/academic-paper-reviewer/
├── SKILL.md                            (主定义)
├── references/
│   ├── review_criteria_framework.md    (7 维度评分标准 + 论文类型专项标准)
│   └── editorial_decision_standards.md (决策矩阵 + 审稿伦理)
```

## Step 4: academic-pipeline — 全流程编排

### 触发方式

| 你说 | 行为 |
|------|------|
| `启动学术全流程 [选题]` | 从研究到论文的完整流程：研究 → 完整性门控 → 写作 → 引用审计 → 评审 → 终稿 |
| `从零开始写论文 [选题]` | 同上，假设无前期材料 |

### 流程总览

```
Stage 1: 深度研究 (deep-research)
  ──→ [用户确认]
    ↓
Stage 2.5: 完整性门控 [强制 — 不可跳过]
  ──→ [PASS / FAIL → 修复重检，最多 3 轮]
    ↓
Stage 3: 论文写作 (academic-paper)
  ──→ [用户确认]
    ↓
Stage 4.5: 引用审计 [强制 — 不可跳过]
  ──→ [PASS / FAIL → 修复重检，最多 3 轮]
    ↓
Stage 5: 同行评审 (academic-paper-reviewer)
  ──→ [Accept → Stage 6 / Minor → 用户修改 → Stage 6 / Major → 用户修改 → 重审 → Stage 6 / Reject → 讨论]
    ↓
Stage 6: 最终输出 + 流程总结
```

### 数据传递

| 阶段 | 产出文件 | 输入来源 |
|------|----------|----------|
| Stage 1 | `stage1_research.md` | 用户选题 |
| Stage 2.5 | `stage25_integrity.md` | Stage 1 产出 |
| Stage 3 | `stage3_draft.md` | Stage 1 + 2.5 产出 |
| Stage 4.5 | `stage45_citation_audit.md` | Stage 3 产出 |
| Stage 5 | `stage5_review.md` | Stage 3 + 4.5 产出 |
| Stage 6 | `final_paper.md` + `process_summary.md` | 所有之前产出 |

文件统一保存在 `<工作目录>/academic-pipeline/` 下。

### 门控规则

| 门控 | 检查内容 | 失败处理 |
|------|----------|----------|
| **Stage 2.5** | 捏造引用？方法捏造？框架锁定？幻觉数据？ | HIGH-WARN 暂停，最多修复 3 轮，否则用户需书面 override |
| **Stage 4.5** | 抽查 5 个引用，验证来源是否支持论断 | 同上 |

### 禁止的状态跳转

- ❌ 不能跳过 Stage 2.5（未经完整性核查就写作）
- ❌ 不能跳过 Stage 4.5（未经引用审计就送审）
- ❌ 不能超过 1 轮 re-revise（无限制循环）

### 文件结构

```
~/.hermes/skills/research/academic-pipeline/
└── SKILL.md  (6 阶段编排定义)
```

### 使用提示

- 每个阶段结束需要你输入「继续」才能进入下一阶段
- 你随时可以要求暂停、返回修改或调整方向
- 完整流程通常需要 2-4 轮交互（取决于审稿轮次）
- 推荐先试小题目跑通流程，再做大题目
