# academic-pipeline 工作流

## 概述

academic-pipeline 是 ARS4Hermes 的四步技能之一，串联 deep-research → academic-paper → academic-paper-reviewer 三个技能，构成从选题到终稿的完整学术写作流水线。

## 触发方式

```
启动学术全流程 [选题]
从零开始写论文 [选题]
```

示例：`启动学术全流程 人工智能对统计学本科教学质量保障的影响`

## 六阶段工作流

```
用户触发「启动学术全流程 X」
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 1: 深度研究                                          │
│ 加载 deep-research 技能 → 多角度检索 → 交叉验证 → 研究报告  │
│ 产出: stage1_research.md                                   │
│ 门控: [用户确认] → 继续 / 修改                             │
└────────────────────────────────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 2.5: 完整性门控 [强制 — 不可跳过]                    │
│ 独立重新验证：                                            │
│  F1: 捏造引用 (抽查 5 个，DOI 验证)                       │
│  F2: 方法捏造 (报告的方法是否实际执行了？)                  │
│  F3: 框架锁定 (是否只呈现单方观点？)                       │
│  F4: 幻觉数据 (数字/统计是否有来源？)                      │
│ 产出: stage25_integrity.md                                │
│ 门控: ALL CLEAR → 继续 / HIGH-WARN → 暂停修复 (最多3轮)   │
└────────────────────────────────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 3: 论文写作                                          │
│ 加载 academic-paper 技能 → 确认论文类型 → 生成大纲         │
│  → 逐章写作 (Anti-Leakage) → 中英双语摘要 → APA7 引用      │
│ 产出: stage3_draft.md                                     │
│ 门控: [用户确认] → 继续 / 修改                             │
└────────────────────────────────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 4.5: 引用审计 [强制 — 不可跳过]                      │
│ 抽查5个引用，逐项验证：                                    │
│  - 来源是否存在？                                          │
│  - 论断与来源是否匹配？                                    │
│  - 有无可疑模式？                                         │
│ 产出: stage45_citation_audit.md                           │
│ 门控: ALL CLEAR → 继续 / HIGH-WARN → 暂停修复 (最多3轮)   │
└────────────────────────────────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 5: 同行评审                                          │
│ 加载 academic-paper-reviewer 技能                          │
│  7 维度加权评分 + 问题分类 + 引用核查 + 反奉承机制         │
│ 产出: stage5_review.md                                    │
│ 决策:                                                     │
│  Accept     → Stage 6                                     │
│  Minor      → 用户修改 → Stage 6                           │
│  Major      → 用户修改 → Re-review → Stage 6              │
│  Reject     → 讨论：重构或中止                              │
│ ✦ 最多 1 轮 re-revise                                     │
└────────────────────────────────────────────────────────────┘
         │
         ▼
┌────────────────────────────────────────────────────────────┐
│ Stage 6: 最终输出                                          │
│  产出: final_paper.md                                     │
│  流程总结: process_summary.md                              │
└────────────────────────────────────────────────────────────┘
```

## 数据传递

| 阶段        | 消费                         | 产出                                  |
| --------- | -------------------------- | ----------------------------------- |
| Stage 1   | 用户选题                       | stage1_research.md                  |
| Stage 2.5 | stage1_research.md         | stage25_integrity.md                |
| Stage 3   | stage1_research.md (门控通过后) | stage3_draft.md                     |
| Stage 4.5 | stage3_draft.md            | stage45_citation_audit.md           |
| Stage 5   | stage3_draft.md (审计通过后)    | stage5_review.md                    |
| Stage 6   | 所有之前产出                     | final_paper.md + process_summary.md |

## 门控规则

| 门控        | 检查内容                | 失败处理                                   |
| --------- | ------------------- | -------------------------------------- |
| Stage 2.5 | 捏造引用、方法捏造、框架锁定、幻觉数据 | HIGH-WARN → 暂停，最多3轮修复，否则用户需书面 override |
| Stage 4.5 | 抽查5个引用，验证来源是否支持论断   | 同上                                     |

## 禁止的跳转

- ❌ 不能跳过 Stage 2.5（未经完整性核查就写作）
- ❌ 不能跳过 Stage 4.5（未经引用审计就送审）
- ❌ 不能超过 1 轮 re-revise（无限制循环）
- ❌ 不能从 Stage 4 直接到 Stage 6（必须经过评审）
